Github
Folder yang dalam git di sebut Repository.
## Agar folder di kenali git, atau folder ingin di jadikan repository , maka Perintahnya $ git init ## untuk menambahkan file ke dalam staging area by file $ git add <nama file> ## untuk menambahkan file ke dalam staging area all file $ git add . ## membatalkan stage area alias di unstage, $ git reset ## cek status file / cek status repo apakah ada file baru atau file modif $ git status ## Commit atau menyetujui segala perubahan $ git commit -m "tulis pesannya disini" ## untuk memasukan configuration, seperti username password github $ git config ## untuk melihat list config git $ git config --list ## config local repo tertentu, repo yang sedang aktif $ git config --local ## untuk meng config seleruh repo yang ada di local git $ git config --global ## config username dan password sesuai akun github $ git config --global user.name "fn14" -> tekan enter
$ git config --global user.email "email@gmail.com" -> tekan enter ## melihat isi branch yang ada $ git branch ## untuk membuat branch $ git branch <nama branch> ## pindah branch $ git checkout <nama branch> ## jika git nya mau di hubungkan dengan github , dan mengikuti repo yang ada di github maka $ git clone <url git nya> *** jika repo nya ada di local dan di github belum ada repo atau github mengikuti repo local, maka cara nya tetap harus bikin repo dulu di github. Bedanya jangan di checklist inialize this repository nya, biarkan kosong, tidak ada readme atau file apa pun. ketika sudah, klik button Create Repository. dan ikuti saja petunjuk / keterangannya nya

## untuk lihat data remote nya $ git remote ## detail data remote nya / detail remote github nya $ git remote -v ## untuk membuat alias graph, jadi ketika mau melihat graph brach commit nya, cukup ketik comaand graph $ alias graph="git log --all --decorate --oneline --graph" ## untuk push data dari git local ke github / remote $ git push ## JIKA ADA CONFLIC ketika PUSH , maka lakukan
1. cek repo github apakah sudah sama dengan repo local yang kita punya $ git fetch 2. ambil full yang ada di remote nya $ git full 3. ketika buka file codingan, maka akan terdapat mark terjadi nya konflik, lalu betul kan secara manual, kalau sudah tinggal git add, commit dan terakhi git push


Docker

## Cek Apakah di local kita ada docker image / melihat images yang sudah kita download > docker images ## cara Run Docker dan jika tidak ada di local, maka automatis akan ambil image ke docker hub > docker run -it ubuntu ## cara Run Docker dengan image version > docker run -it ubuntu:18.04 ## Melihat daftar Containner yang running > docker container list ## Melihat daftar container yang running maupun tidak > docker container list --all ## cara liat list docker yang sedang running > docker ps ## cara liat docker yang pernah di jalanin > docker ps -a ## cara menjalankan container (nama container) > docker container start mongoserver1 ## cara menjalankan container lebih dari satu > docker container start mongoserver1 mongoserver2 ## dan cara menjalankan docker yang pernah di stop > docker container start af761cd2ca3b (id docker nya) ## cara masuk kembali ke sh nya docker image ( ubuntu ) > docker exec -it af761cd2ca3b sh ## cara stop docker ( bisa langsung tutup shell / cmd nya ) atau ketik > docker container stop 7a6b48037b53 ## cara stop docker ( bisa langsung tutup shell / cmd nya ) atau ketik > docker rm 7a6b48037b53 ## Create container dari image yang kita punya > docker container create --name mongoserver1 -p 8080:27017 mongo:4.4.1 ## cara membuat and run port forward agar port docker bisa di akses port bawaan engine (-p ) di docker ada konsep, host dan container, agar host bisa mengakses container maka gunakan port forward > docker run --name some-postgres -p 5432:5432 -e POSTGRES_PASSWORD=dockerpostgres -d postgres ## hapus container > docker container rm mongoserver1 ## hapus container lebih dari 1 > docker container rm mongoserver1 mongoserver2 ## cara mengganti tag > docker tag ubuntu:18.04 ubuntu-belajar:v1 ## push image docker kita ke hub docker ( namaidhubdocker/nama image nya )
